Kubia demo

"Kubia" stands in for any SaaS integrating PortixOne. This walks through Milestone 3's actual target flow: register a business → connect a printer (pair) → choose which one → make a sale → print the receipt. Requires the PortixOne Runtime running (npm run dev in runtime/).

2. Connect a printer

Approve this from the PortixOne Tray's Pairing Requests menu (right-click the tray icon). For scripted testing without the Tray, the Runtime's admin key also works: curl -X POST http://127.0.0.1:17321/pairing/approve -H "Content-Type: application/json" -H "x-portix-api-key: <admin key>" -d "{\"code\":\"...\"}"
